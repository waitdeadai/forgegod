# AGENTS.md
Scope: entire Tarot showcase repository.

This repository exists to benchmark ForgeGod, not to bypass it.

## Mission
- Build the Tarot showcase from zero.
- Do not reuse application code from the archived legacy Tarot project.
- Treat the product as a harness benchmark: if the result is weak, improve ForgeGod.

## Rules
- Read `docs/README.md`, `docs/PRD.md`, `docs/ARCHITECTURE.md`, `docs/RUNBOOK.md`, `DESIGN.md`, and `taste.md` before writing code.
- Keep the visual language aligned with ForgeGod branding: cyan, gold, dark surfaces, pyramid/halo/`1` symbolism where appropriate.
- Optimize for mobile-first responsive behavior.
- Free-only v1: no payments yet.
- Every implementation task must leave clear verification evidence.
- Prefer production-grade defaults over flashy shortcuts.
- Run current-year web research before meaningful architecture, dependency, animation, or deployment choices.
- Do not reuse app code from the prior tarot showcase. This cleanroom is implementation-only from docs and harness execution.
- Use ForgeGod itself for planning, research, implementation, testing, and review. Human edits here are limited to harness contracts and execution setup.

## Delivery Standard
- The app must be defensible as a public ForgeGod showcase.
- If ForgeGod gets blocked, document the blocker in repo docs and fix the harness upstream when possible.
